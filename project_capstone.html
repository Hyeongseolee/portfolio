Precision Trajectory Generation & Real-Time Web Control System

Bridging User Creativity with Robot Kinematics via Full-Stack Architecture

Python (PyQt & Flask) Trajectory Generation Robot Kinematics WebSockets

🔹Context & Objective

Overview: A multi-axis robotic manipulator system engineered to replicate the delicate, non-linear fluid-flow and wrist-snap techniques of human baristas.

Objective: To build an automated latte art system that allows non-expert users to draw custom patterns on a GUI, which the robot instantly translates into smooth, collision-free physical trajectories, while maintaining an accessible and robust real-time control architecture.

🔹Technical Implementation

1. Trajectory Generation Algorithm

  • Implemented continuous path interpolation (e.g., Spline/Bézier profiles) in `Python` to convert discretized pixel coordinates from the UI into time-dependent position, velocity, and acceleration profiles.
  • Optimized the velocity profile to bound the maximum joint acceleration, significantly reducing mechanical jerk and preventing milk spillage during high-curvature drawing motions.

2. HMI UI & Full-Stack Transition

  • Initially developed a standalone desktop application using `Python (PyQt)` integrating a responsive drawing canvas and safety dashboard.
  • Transitioned the architecture to a universal `HTML/JS` web UI powered by a `Flask` backend server to overcome hardware integration constraints.
  • Implemented `Flask-SocketIO` (WebSockets) to broadcast the robot's real-time position, joint status, and task progress (Progress Bar) directly to the web browser.

3. System Integration & Communication

  • Designed REST APIs using Flask to seamlessly map web UI user inputs to underlying UR3e robot hardware commands.
  • Handled multi-motor synchronization via serial communication, mapping generated profiles directly to embedded motor controllers with deterministic cycle times.

🔹Deliverables & Troubleshooting

Challenge 1: Communication Lag During early testing, sharp turns in drawing patterns caused communication lag and jerky motor responses.

Solution 1: Applied a window-based data buffering technique and integrated a look-ahead smoothing algorithm in Python, ensuring steady packet transmission and a 105° sharp-turn precision alignment.

Challenge 2: App Limitations The initial Flutter/PyQt approach faced limits in real-time UR3e integration and screen sharing.

Solution 2: Pivoted to a full-stack HTML/JS + Flask WebSocket architecture, resulting in a highly responsive, real-time barista robot system with a custom cafe order queue.
